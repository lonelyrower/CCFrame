# 交互菜单重新设计说明

## 🎯 改进目标

基于用户反馈，重新设计交互菜单，解决以下问题：
1. ❌ 菜单结构混乱，选项过多
2. ❌ 安装和更新各有两个选项（源码/镜像），重复
3. ❌ "切换部署模式"功能不清晰
4. ❌ 颜色代码显示为乱码

## 📊 菜单对比

### 旧菜单（13个选项）

```
请选择要执行的操作：
  1) 初始化部署（自动选择模式）
  2) 初始化部署（强制镜像模式）    ← 重复
  3) 更新（源码构建）               ← 重复
  4) 更新（镜像部署）               ← 重复
  5) 切换部署模式                   ← 功能不明确
  6) 启动
  7) 重启
  8) 停止
  9) 查看状态
 10) 查看日志
 11) 生成或修复 .env
 12) 健康检查
 13) 卸载
  0) 退出
```

**问题：**
- 安装和更新各有 2 个选项（源码/镜像）
- 用户需要提前决定用哪种方式
- "切换部署模式"是切换镜像/源码还是HTTP/HTTPS？

### 新菜单（11个选项）

```
CCFrame 管理菜单

─── 部署管理 ───
  1) 初始化部署（全新安装）        ← 合并，选后再问
  2) 更新部署（保留数据）          ← 合并，选后再问
  3) 切换 HTTPS 模式（IP/域名/Cloudflare）  ← 功能明确

─── 服务管理 ───
  4) 启动服务
  5) 停止服务
  6) 重启服务
  7) 查看状态
  8) 查看日志

─── 配置管理 ───
  9) 修复环境配置 (.env)
 10) 健康检查

─── 系统维护 ───
 11) 卸载系统
  0) 退出
```

**改进：**
- ✅ 选项减少到 11 个（-15%）
- ✅ 清晰的功能分组
- ✅ 安装/更新各一个选项，选后再询问方式
- ✅ 功能命名准确无歧义

## 🔄 交互流程改进

### 旧流程：提前决定

```
用户 → 菜单：选择"初始化部署（强制镜像模式）"
       ↓
     直接执行镜像部署
```

**问题：** 用户需要提前了解镜像和源码的区别

### 新流程：引导式选择

```
用户 → 菜单：选择"初始化部署"
       ↓
     系统：显示详细说明
       ↓
     ━━━━━━━━━━━━━━━━━━━━━
     CCFrame 部署方式选择
     ━━━━━━━━━━━━━━━━━━━━━
     
     1) 镜像部署（推荐）
        ✓ 使用预构建的 Docker 镜像
        ✓ 部署时间：3-5 分钟
        ✓ 内存需求：最低 512MB
        ✓ 适合：生产环境快速部署
     
     2) 源码构建
        • 从 GitHub 克隆源码并本地构建
        ✓ 部署时间：15-30 分钟
        ✓ 内存需求：至少 2GB
        ✓ 适合：需要自定义修改代码
     
     请选择 [1/2] (默认: 1):
       ↓
     用户：看清楚区别后再选择
```

**优势：**
- ✅ 符合用户思维：先决定"做什么"，再决定"怎么做"
- ✅ 显示详细说明，帮助用户做出正确选择
- ✅ 推荐选项明确（镜像部署）

# 显示为：

  \033[0;32m1) 🚀 镜像部署（推荐）\033[0m  ← 乱码！

  1) 镜像部署（推荐）  ← 绿色，无乱码
```

## 📖 功能说明：切换 HTTPS 模式

### 重命名的原因

- **旧名称**：`cmd_switch_mode` / "切换部署模式"
- **歧义**：是切换镜像/源码？还是切换 HTTP/HTTPS？
- **新名称**：`cmd_switch_https_mode` / "切换 HTTPS 模式"
- **明确**：只改变访问方式（IP/域名/Cloudflare），不重新部署

### 功能说明

```
━━━━━━━━━━━━━━━━━━━━━━━━━
   切换 HTTPS 访问模式
━━━━━━━━━━━━━━━━━━━━━━━━━

当前部署模式：ip

请选择新的访问模式：
  1) IP 模式（HTTP，仅服务器 IP 访问）
  2) Let's Encrypt（自动申请免费 SSL 证书）
  3) Cloudflare（使用 CDN 提供的 HTTPS）
```

**操作：**
- 只修改 nginx 配置
- 只修改 .env 中的 HTTPS 相关变量
- 重启 nginx 服务（不重启应用）
- 不会丢失数据

**适用场景：**
- 初次安装时选择了 IP 模式，后来想启用 HTTPS
- 从 Let's Encrypt 切换到 Cloudflare
- 从 Cloudflare 切换回 IP 模式（降级）

## 🔧 命令行兼容性

# 命令别名支持

```

## 🎓 设计原则

1. **简化选择** - 先决定目标，再决定方法
2. **引导式交互** - 提供详细说明，帮助用户选择
3. **功能分组** - 按用途分类，易于查找
4. **命名明确** - 避免歧义，一目了然
5. **视觉友好** - 正确的颜色，清晰的布局
