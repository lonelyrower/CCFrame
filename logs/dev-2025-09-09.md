# 开发日志（2025-09-09）

## 今日概览
- 引入 Toast UI 高级图片编辑，并与基础编辑统一为“编辑版本”保存流程。
- 扩展 Clipdrop 能力：替换背景、画布扩展（Uncrop）、重光照、去文字，并打通前后端参数传递。
- 将 /api/ai/enhance 改为入队处理（BullMQ），统一由 Worker 执行；新增每用户 15 分钟 20 次限流。
- 修复多处类型与响应体问题，确保 TypeScript 全量通过与服务端流式返回稳定。
- 优化文案，移除“真·”类表述，保持统一风格。

## 主要改动
- 高级/基础编辑
  - 新增 AdvancedEditor（Toast UI 包装，动态加载）：`components/editor/advanced-editor.tsx`
  - 保留 SimpleEditor（裁剪/旋转/翻转/缩放）：`components/editor/simple-editor.tsx`
  - 基础编辑页支持切换高级/基础并统一保存：`app/admin/basic-editor/page.tsx`
  - 新增保存接口并生成 EditVersion：`POST /api/edit/save` → `app/api/edit/save/route.ts`
  - 全局引入样式（layout 顶层 import）：`app/layout.tsx`
- Clipdrop 扩展（lib/ai-providers.ts）
  - 新增 `replaceBackground` / `uncrop` / `relight` / `removeText`，统一 Buffer→Blob 处理
  - `AIImageProcessor` 支持新任务类型与 auto provider 选择
- AI 任务与队列
  - `/api/ai/enhance`：入队（BullMQ），不再路由内直接执行；加入限流（lib/rate-limit.ts）
  - `jobs/queue.ts`：实现各 AI 任务的 Worker 处理与 EditVersion 写入
  - `types/index.ts`：补充 JobType 枚举项（REPLACE_BACKGROUND/UNCROP/RELIGHT/REMOVE_TEXT）
- 类型与稳定性
  - `app/api/image/*`：以 `Uint8Array` 返回响应体，修复 Next/TS 对 Response 的兼容问题
  - `lib/smart-albums.ts`：修复 AND/OR 条件构造的类型与 push 问题
  - `lib/auto-tag.ts`：显式 map 参数类型
  - `lib/ai-services.ts` 标记弃用（改用 ai-providers）
- 文案与 UI
  - AI 设置页：将“启用 真·AI放大 与 真·去背景 功能。”改为“启用 AI 放大与去背景功能。”
  - AI 工具页加入替换背景/重光照/Uncrop/去文字任务卡与参数弹窗

## 运行与验证
- 依赖安装（已用镜像安装）：
  - `npm config set registry https://registry.npmmirror.com`
  - `npm i tui-image-editor tui-color-picker fabric`
- 开发启动：`npm run dev`
- Worker 启动（可选，本地验证队列）：`START_WORKERS=true npx tsx jobs/worker.ts`
- 类型检查：`npm run type-check`（已通过）

## 风险与注意
- Toast UI 样式在 layout 顶层引入；如未安装依赖会导致构建失败（当前已安装）。
- Clipdrop/OPENAI/Gemini 的 Key 建议仅在后台设置或环境变量中配置；日志已尽量避免泄露。
- 限流为 Redis 依赖，开发环境 Redis 不可用时 fail-open，不阻塞请求。
